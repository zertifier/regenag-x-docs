# Ocean Protocol, mercado e IPFS

Ocean Protocol habilita mercados de datos y de servicios de IA con contratos inteligentes y un sistema de permisos flexible. Los activos (datasets, algoritmos) se publican con metadatos, políticas y precios; las referencias a los ficheros pueden residir en IPFS u orígenes privados.

Componentes clave:
- Tokens y transacciones: cada activo puede tokenizarse (ERC-20) para gobernanza y tarificación; el consumo se paga por transacción on-chain.
- Ocean Node: nodo que orquesta publicación, permisos, Compute-to-Data y registro en blockchain.
- Sistema distribuido: los datos permanecen donde están, mientras el cómputo se desplaza al dato.
- Ocean Enterprise: mercado análogo/derivado alineado con Gaia-X para federar catálogos y monetización, interoperando con credenciales verificables.

Ventajas:
- Monetización transparente y auditable.
- Separación entre metadatos públicos y datos privados.
- Integración con Compute-to-Data para proteger la soberanía del dato.
